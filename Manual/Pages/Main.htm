News in 2.50 from 2.43

Final Stable Release

The next version of amai has finally reached a status worthy of a stable release. This has been a major project which resulted in massive changes to the underlying code. This is the first major update to amai since January 2005 where the new year edition was released. I would like to give my thanks to all the play testers of the betas who helped make this final release possible. You guys provided a wealth of valuable information and suggestions that made this possible. See you all again for the development of 2.51 ;) and have fun playing warcraft.

This massive release features a wealth of improvements and bug fixes. Due to the amount of bugs and changes made it is unfeasible to list every single possible change ever made so below lists most of the major improvements from 2.43. If your upgrading your custom AMAI read the developer edition change log below for help but due to the amount of changes it will take a bit of work to get working.

NOTE: Certain languages have not had all strategies totally translated yet. Please visit the forums for information about this.

AMAI ENGINE IMPROVEMENTS

  • Improved defeat conditions where profiles will give up or fight to the end.
  • The Start Lag of 2.43 removed and a whole new pathing calculation system is in place which is fast and effective without the lag. Also in turn fixes bugs of trying to use unpathable neutrals.
  • Loads of Strategy improvements and tweaks to further enhance amai.
  • A much more efficient ghoul lumber harvesting system.
  • Small tweaks to try and improve overall team play with each other.
  • The way gold mines are chosen have been improved to choose out of the the closest mines instead of sometimes always picking mines near enemy locations.
  • AMAI can now make much more effective use of Millitia expansion, Tower rushes and ancient expansions.
  • Undead now can build necropolis at expansions.
  • Mana fountains can now be used by AMAI.
  • Improved the army tracking systems to be more accurate and less error prone.
  • Improved retreat systems with more aggressive profiles less likely to retreat in combat.
  • Major new micro management of units with correct usage of racial shops to heal units.
  • Improved focus fire systems which also includes focusing fire on enemy structures and wards.
  • Fixed many teleportation issues
  • Fixes to the core attack systems of AMAI
  • Fixed multiple zepplin usage bugs
  • Much improved hero selection
  • Portuguese and Norwegian languages now included in AMAI
  • Multiple problems fixed for the ROC version of AMAI
  • A new AMAI version allowing play against both AMAI and the original blizzard ai at once.

  • Improved calculation speeds of pathing systems to fix issues on certain maps where certain neutral were not calculated unpathable early enough.

  • Fixed an issue where invisible units could be used to exploit amai into defending its town

  • Wheel of Chaos map is now confirmed fixed as to AMAI being able to play fine on the map

  • Tower rush is now more likely to occur at nighttime to take advantage of decreases sight.
  • Chinese language has been disabled due to needs proper translation.

AMAI DEVELOPER IMPROVEMENTS

  • AMAI common.ai now nativly supports scripts made in the world editor and original blizzard ai scripts. ( There is only one possible issue that can occur with pure custom scripts please see the AMAI vs AI section for details )
  • AMAI simple edition now uses grimoire to run AMAI which allows support for running warcraft with parameters and enabling the debugging tools.
  • Debugging tools of AMAI have been greatly improved with a new system that displays debug info above the unit in question instead of spamming text screen with messages causing a slow down.
  • minimum and maximum is now superseded by Min and Max function calls.
  • DisplayToAllDebug function is now called Trace but no functionality has changed.
  • Added 'mine' type that determines version gold mines to standardunits.txt.
  • Added 'lumber' type that determines the drop off point structures so they are built next to trees to standardunits.txt
  • Added 'zigg' type to determine structures that are ziggurats to standardunits.txt
  • DefendsTownCond now has an extra parameter max_dist
  • Instead of using the disable retreat setting a new text file called standardaisettings has been created that allows you to modify all the blizzard controlled settings (ones that appear in ai editor) including disabling unit retreating etc. You can also see which settings are controlled by AMAI and changing them will not make a blind bit of difference.
  • A new advanced upgrade function has been added.
    BuildAdvUpgrade([quantity], [upgrade], [start tier],[unit count],[start unit count],[tier start count bonus],[tier prority bonus],[base priority])
    This function will build upgrade when unit count greator then the start count with a prority of (base priority * unit count). It only takes affect at the designated start tier and then builds at first 1 only, then increases every tier till it reaches the quantity level specified in the function. This function adds dynamic approach to upgrades so AMAI does upgrades based on the number of units.
  • Added a compile AMAI vs AI version button. This compiles the TFT version scripts into the AMAI vs AI executable. This means tweaks you make to AMAI you can then test out your own tweaks easily against the standard AI without doing the long route of using an MPQ tool manually. (And to be honest it was getting annoying for myself)
  • Improvements to the heroes.txt. Has extra first hero, second hero, third hero bonuses for each hero. This provides more power for hero selection to make more accurate accurate hero choices by AMAI.
  • MassHealing.txt file has been replaced with the new HealingItems.txt file. There are 4 types of healing items available. HEALINGITEM_MASS, HEALINGITEM_SINGLE, MERCHANTITEM_MASS,HEALINGITEM_HERO
  • New profile surrender value which has settings between 0 and 4. 0 is the default blizzard value in this case never gives up. While 4 means the profile gives up extremly easy. 1 is the original surrender system for amai.
  • A Global setting has been added which provides whether or not this amai version has heroes. Turning off will not prevent heroes from being trained as such if its in there strategy but it makes attack assaults not wait upon a hero.
  • A Global Setting added that turns off the blizzard controlled retreating. This is only useful for amai's that are used with units of extremly low health where such retreating style is not effective.
  • A global Setting to set all computers to have the name "Computer" instead of their profile name.
  • Fixed a bug related to having an empty UnitEquivelence.txt file which caused the compilor to complain about a missing endif.
  • Maps that use the set custom value of unit native or coressponding trigger are now compatible with amai.
  • Level Strength Map text file has become redundent and is no longer used due to enhanced strength calculations.
  • 3 new global settings have been added for the strength bonus that is added depending on if a creep camp is minor or major.

We have made some major improvements to amai code as stated above. Therefore to allow your customized 2.43 amai to work in this new version you will need to make the following changes.

  • The "SetBuildUnit" "SetBuildItem" "SetBuildUpgr" commands are now "BuildUnit" "BuildItem" "BuildUpgr" respectively.
  • The reference to a hero e.g. hero_id[1] or hero_id[2] is now references as hero[1] or hero[2]
  • Then click the applytablechanges.pl to create the nessecary variable additions to tables. (Note some global settings do not get added correctly so you will have to compare manually)
  • These only affect AMAI code. The reason for the changes is to allow the standard blizzard ai codes to work. Therefore "SetBuildUnit" would call the standard build code which will cause issues if called in AMAI scripts.
